版本: v1.0.0

日期: 2025年5月

开源项目

1\. 项目概述

1.1 项目背景

博物馆作为文化遗产保护的重要场所，其安全管理至关重要。传统的人工巡检方式存在效率低、覆盖不全、记录不规范等问题。本项目旨在开发一套开源的智能巡检系统，帮助博物馆实现巡检工作的数字化、智能化管理。

1.2 项目目标

-   提供完整的巡检任务管理功能，支持日常巡检、专项巡检、临时巡检等多种类型

-   支持扫码打卡、NFC打卡、GPS定位打卡等多种打卡方式

-   实现问题工单的闭环管理，从发现到整改全程可追溯

-   提供数据统计与分析功能，辅助管理决策

-   开源免费，支持私有化部署，保护数据安全

1.3 适用范围

本系统适用于各类博物馆、展览馆、美术馆、纪念馆等文化场馆的巡检管理工作，也可扩展应用于景区、工厂、园区等场景。

4\. 数据库设计

4.1 核心表结构

4.1.1 巡检任务表 (patrol\_tasks)

  -------------- -------------- --------------------- -----------------------------------------
  **字段名**     **类型**       **约束**              **说明**
  id             BIGSERIAL      PK                    主键
  name           VARCHAR(100)   NOT NULL              任务名称
  type           VARCHAR(20)    DEFAULT \'daily\'     任务类型: daily/special/temp
  frequency      JSONB          \-                    频率配置
  route\_id      BIGINT         FK                    关联路线ID
  assignee\_id   BIGINT         FK                    负责人ID
  status         VARCHAR(20)    DEFAULT \'pending\'   状态: pending/running/completed/overdue
  plan\_time     TIMESTAMP      NOT NULL              计划执行时间
  -------------- -------------- --------------------- -----------------------------------------

4.1.2 巡检路线表 (patrol\_routes)

  -------------- -------------- ---------- ------------------------------
  **字段名**     **类型**       **约束**   **说明**
  id             BIGSERIAL      PK         主键
  name           VARCHAR(100)   NOT NULL   路线名称
  description    TEXT           \-         路线描述
  museum\_zone   VARCHAR(50)    \-         博物馆区域(展厅/办公区/库房)
  -------------- -------------- ---------- ------------------------------

4.1.3 巡检点位表 (route\_points)

  -------------- --------------- ---------- ------------
  **字段名**     **类型**        **约束**   **说明**
  id             BIGSERIAL       PK         主键
  route\_id      BIGINT          FK         关联路线ID
  name           VARCHAR(100)    NOT NULL   点位名称
  qr\_code       VARCHAR(100)    UNIQUE     二维码标识
  latitude       DECIMAL(10,8)   \-         纬度
  longitude      DECIMAL(11,8)   \-         经度
  check\_items   JSONB           \-         检查项配置
  -------------- --------------- ---------- ------------

4.1.4 巡检记录表 (patrol\_records)

  ---------------- ------------- ---------- ------------------------------
  **字段名**       **类型**      **约束**   **说明**

  task\_id         BIGINT        FK         关联任务ID
  point\_id        BIGINT        FK         关联点位ID
  inspector\_id    BIGINT        FK         巡检人员ID
  status           VARCHAR(20)   \-         状态: normal/abnormal/missed
  check\_time      TIMESTAMP     NOT NULL   打卡时间
  check\_method    VARCHAR(20)   \-         打卡方式: qr\_code/nfc/gps

5\. API设计

5.1 API规范

-   基础路径: /api/v1

-   认证方式: Bearer Token (JWT)

-   数据格式: JSON

-   编码: UTF-8

5.2 核心接口列表

5.2.1 巡检任务接口

  ---------- ---------------------------- ------------------------------------
  **方法**   **路径**                     **说明**
  GET        /api/v1/tasks                获取巡检任务列表（支持筛选、分页）
  POST       /api/v1/tasks                创建巡检任务
  GET        /api/v1/tasks/:id            获取任务详情
  PUT        /api/v1/tasks/:id/start      开始巡检
  PUT        /api/v1/tasks/:id/complete   完成巡检
  ---------- ---------------------------- ------------------------------------

5.2.2 巡检打卡接口

  ---------- ---------------------------- --------------------------
  **方法**   **路径**                     **说明**
  POST       /api/v1/records              提交巡检记录（扫码打卡）
  POST       /api/v1/records/:id/photos   上传巡检照片
  GET        /api/v1/records              获取巡检记录列表
  ---------- ---------------------------- --------------------------

5.2.3 路线管理接口

  ---------- --------------------------- ------------------
  **方法**   **路径**                    **说明**
  GET        /api/v1/routes              获取巡检路线列表
  POST       /api/v1/routes              创建巡检路线
  PUT        /api/v1/routes/:id          更新巡检路线
  GET        /api/v1/routes/:id/points   获取路线点位列表
  ---------- --------------------------- ------------------

5.2.4 工单管理接口

  ---------- ------------------------------ --------------
  **方法**   **路径**                       **说明**
  GET        /api/v1/tickets                获取工单列表
  POST       /api/v1/tickets                创建工单
  PUT        /api/v1/tickets/:id/assign     指派处理人
  PUT        /api/v1/tickets/:id/complete   完成工单
  ---------- ------------------------------ --------------

5.2.5 数据统计接口

  ---------- -------------------------- --------------
  **方法**   **路径**                   **说明**
  GET        /api/v1/stats/overview     巡检概览统计
  GET        /api/v1/stats/completion   完成率统计
  GET        /api/v1/stats/abnormal     异常分析统计

6\. 部署方案

6.1 环境要求

  ---------- ------------------------------------------------
  **组件**   **最低配置要求**
  服务器     2核CPU / 4GB内存 / 50GB硬盘
  操作系统   Linux (Ubuntu 20.04+ / CentOS 7+) / Docker环境
  Docker     Docker 20.10+ / Docker Compose 2.0+
  网络       开放80/443端口，内网可访问数据库端口
  ---------- ------------------------------------------------

6.2 Docker部署

推荐使用Docker Compose一键部署：

> git clone https://github.com/your-org/museum-patrol.git
>
> cd museum-patrol
>
> cp .env.example .env
>
> \# 编辑 .env 配置数据库密码等
>
> docker-compose up -d

6.3 配置说明

主要配置项（configs/config.yaml）：

-   server.port: 服务端口，默认8080

-   database: PostgreSQL连接配置

-   redis: Redis连接配置

-   jwt.secret: JWT密钥，生产环境必须修改

-   storage: MinIO文件存储配置

7\. 安全设计

7.1 认证与授权

-   采用JWT进行身份认证，Token有效期可配置

-   基于角色的权限控制(RBAC)，支持管理员、巡检员、普通用户等角色

-   敏感操作记录审计日志

7.2 数据安全

-   密码采用bcrypt加密存储

-   支持HTTPS加密传输

-   数据库定期备份机制

-   支持私有化部署，数据不出馆

8\. 扩展性设计

8.1 插件化架构

系统预留插件接口，支持扩展：

-   自定义打卡方式插件（如蓝牙信标）

-   第三方通知渠道插件（如企业微信、钉钉）

-   数据导出格式插件

8.2 微服务演进

单体架构设计时已考虑微服务拆分路径：

-   API网关层独立部署

-   各业务服务可独立拆分为微服务

-   预留Protocol Buffers定义，支持gRPC通信
